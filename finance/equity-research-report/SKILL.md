# 业务驱动型深度研报生成器

## 语言要求

**所有输出使用简体中文。** 股票代码、公司英文名、数字及百分比保留原格式（如 NVDA、+12.4%、4.2×）。
数据来源名称（Finnhub、Macrotrends、yfinance）可保留英文。其余全部简体中文。
**禁用 ASCII 字符画/流程图**，统一用 Markdown 表格 + 列表呈现（便于粘贴到 Notion）。

---

## 这份研报的定位

本研报**面向中长线投资者**，核心回答一个问题：**「这是不是一门好生意，现在的价格值不值得参与？」**

四大模块按重要性排序，**模块 2（业务深度分析）是全篇核心，应投入最多篇幅与思考**：

| 模块 | 作用 | 篇幅权重 |
|------|------|---------|
| 1. 公司快照 | 30 秒了解公司是谁 | 10% |
| **2. 业务深度分析** | **回答「是不是好生意」——本研报的灵魂** | **45%** |
| 3. 财务体检 | 用数字验证业务质量 | 25% |
| 4. 估值判断 | 回答「现在值不值得买」 | 20% |

> ⚠️ 切忌把研报写成数据罗列。模块 2 必须有**业务洞察**：解释商业模式背后的逻辑、护城河为什么成立、增长能不能持续。数据是论据，洞察才是结论。

## 执行流程

### Step 0 — 确认 Ticker

若用户给出代码（如「帮我写 PLTR 的研报」），直接执行。若未给出，询问一次。

---

### Step 1 — 并行抓取基础数据

并行调用：
```
mcp__finnhub__get_profile(symbol="TICKER")
mcp__finnhub__get_quote(symbol="TICKER")
mcp__finnhub__get_news(symbol="TICKER")
```

从 profile 提取：公司名、行业、市值、流通股数、Beta、国家、交易所。
从 quote 提取：当前股价、当日涨跌、（若有）52周高低。
从 news 提取：近 30 天关键业务动态（用于后续催化剂与业务进展判断）。

---

### Step 2 — 公司快照（模块 1）

用最精炼的方式让读者 30 秒看懂这家公司：
- **一句话业务定义**：用「卖什么给谁、靠什么收费」描述（电梯演讲）。
- 基本信息：市值、行业、上市时间、总部、当前股价、52周区间内位置。

> 这一模块要短。不要展开，展开留给模块 2。

---

### Step 3 — 业务深度分析（模块 2 · 核心）

**这是本研报最重要的部分，必须深挖。** 按以下五个子模块逐一展开，每个子模块都要给出**洞察而非仅数据**。

#### 3.1 商业模式拆解（How it makes money）
- 公司**具体如何赚钱**？（一次性销售 / 订阅 / 交易抽佣 / 广告 / 授权 / 硬件+服务）
- **单位经济模型**：每获取一个客户/卖出一个单位，收入与成本结构如何？是否有规模效应？
- 收入的**质量**：经常性收入（recurring）占比多少？客户黏性如何？
- **关键经营指标**（按行业选取）：SaaS 看 ARR/NRR/客户留存；消费看同店增长/复购；平台看 GMV/take rate；半导体看产能/ASP。

#### 3.2 收入构成拆解（Revenue breakdown）
- 按**业务分部**拆解收入占比与各自增速（用 Macrotrends / 财报分部数据）。
- 按**地域**拆解（识别地缘/汇率风险）。
- 找出**增长引擎**（哪块在快速增长）和**现金牛**（哪块稳定供血）。

用表格呈现：

| 业务分部 | 收入占比 | 近一年增速 | 角色 |
|---------|---------|-----------|------|
| 分部A | XX% | +XX% | 增长引擎 |
| 分部B | XX% | +X% | 现金牛 |

#### 3.3 护城河分析（Moat · 最关键的判断）
逐一评估五类护城河是否成立，**给出理由，不要只贴标签**：

| 护城河类型 | 是否具备 | 强度 | 论据 |
|-----------|---------|------|------|
| 网络效应 | 是/否 | 强/中/弱 | 具体说明 |
| 转换成本 | 是/否 | … | 客户迁移的代价 |
| 成本优势 | 是/否 | … | 规模/工艺/区位 |
| 无形资产 | 是/否 | … | 品牌/专利/牌照 |
| 规模效应 | 是/否 | … | 是否赢家通吃 |

**结论**：用 1-2 句话定性这家公司的护城河整体是「宽 / 窄 / 几乎没有」，以及护城河是在**变宽还是变窄**（趋势比现状更重要）。

#### 3.4 行业空间与渗透率（TAM）
- 当前所处行业的**总可寻址市场 TAM** 有多大？
- 公司目前的**市占率/渗透率**是多少？还有多少增长空间？
- 行业本身是**扩张、成熟还是衰退**？（这决定了天花板）

#### 3.5 竞争格局（Competitive landscape）
- 列出 2-4 个主要竞争对手，对比规模与差异化定位。
- 这家公司的**相对优势与软肋**是什么？
- 行业竞争是在**加剧还是缓和**？是否有新进入者/替代技术的威胁？

竞争对比表：

| 公司 | 市值 | 核心差异 | 相对本公司 |
|------|------|---------|-----------|
| 本公司 | — | — | — |
| 对手1 | $XXB | … | 更强/更弱在哪 |

> **模块 2 收尾**：用一段话总结「**这是不是一门好生意**」——综合商业模式质量、护城河、行业空间三者，给出明确判断。

---

### Step 4 — 财务体检（模块 3）

用数字**验证**模块 2 的业务判断。好生意应该在财报上留下痕迹（高毛利、扩张的利润率、强现金流）。

获取近 3 年数据（`WebFetch` Macrotrends / stockanalysis.com）：

| 指标 | 3年前 | 2年前 | 最新 | 趋势解读 |
|------|-------|-------|------|---------|
| 营业收入 | | | | 增速是否维持 |
| 毛利率 | | | | 是否反映定价权 |
| 营业利润率 | | | | 经营杠杆是否显现 |
| 净利率 | | | | |
| ROE | | | | 资本回报质量 |
| 自由现金流 (FCF) | | | | 是否赚的是真钱 |
| 净现金/净债务 | | | | 资产负债表健康度 |

**财务结论**：财务数据是否**印证了模块 2 的业务故事**？（例：若声称有定价权，毛利率应稳定或上行；若声称规模效应，利润率应随收入扩张而提升。）若数据与故事矛盾，明确指出。

---

### Step 5 — 估值判断（模块 4）

回答「现在的价格值不值得参与」。

**5.1 相对估值**——当前倍数 vs 行业 vs 自身历史：

| 估值指标 | 当前 | 行业均值 | 自身5年区间 | 判断 |
|---------|------|---------|-----------|------|
| P/E (TTM) | | | | 偏贵/合理/便宜 |
| P/S | | | | |
| EV/EBITDA | | | | |
| PEG | | | | 兼顾增长后是否划算 |

**5.2 绝对估值（可选）**——若需要内在价值，提示可调用 `dcf-valuation` 技能做三情景 DCF。

**5.3 估值结论**：综合相对+绝对，给出当前股价**偏贵 / 合理 / 低估**的判断，以及**大致合理价格区间**。

---

### Step 6 — 综合结论

用一段话收束全篇，必须包含：
- **一句话投资逻辑**（多头核心论点）。
- **最大风险**（空头核心论点）。
- **明确结论**：买入 / 观察 / 回避，附理由。

## 特殊情况处理

- **业务分部数据缺失：** 用 `WebSearch` 查公司最新 10-K/年报的 segment 披露；仍无则按主营定性描述并标注「无细分披露」。
- **早期/亏损公司（FCF 为负）：** 财务体检改看收入增速、毛利率趋势、现金消耗速度（runway）；估值改用 P/S 与 EV/Sales。
- **金融股：** 商业模式聚焦净息差/费率，财务体检看 ROE、ROA、拨备；估值用 P/B 而非 P/E。
- **强周期股（能源/材料/半导体）：** 财务体检需跨越完整周期看，标注当前处于周期哪个阶段。
- **数据抓取失败：** 切换 `WebSearch` 兜底，并在报告中标注数据来源与时效。
- **用户只想要某一模块：** 允许单独输出（如「只要业务分析」就只跑 Step 3）。

## 与其他技能的协作

- 需要**精确内在价值** → 调用 `dcf-valuation`（三情景 DCF）。
- 需要**最新财报电话会解读** → 调用 `earnings-call-analyzer`。
- 需要**买卖时机/仓位评分** → 调用 `seven-dimension-filter`。

本技能聚焦「这是不是一门好生意」的中长线基本面判断，时机与仓位交给上述技能。
